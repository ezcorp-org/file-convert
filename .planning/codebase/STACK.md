## Configuration

**Environment:**
- `.env.example` - Template for environment variables (minimal setup)
- `VITE_PUBLIC_URL` - Public URL for SEO purposes (default: `http://localhost:5173`)
- No backend API required - all conversions happen client-side

**Build:**
- `svelte.config.js` - SvelteKit configuration with Vercel adapter
- `vite.config.ts` - Vite build configuration with worker path aliases
- `tsconfig.json` - TypeScript compiler options (strict mode enabled)
- `.nvmrc` - Node version 20.17.0
- `playwright.config.ts` - E2E test configuration
- `vitest.config.ts` - Unit test configuration

## Platform Requirements

**Development:**
- Node.js 20.x (or Bun as alternative runtime)
- Port 5173 for development server
- Web Worker support in browser
- WebAssembly (WASM) support for optimized conversions

**Production:**
- Deployment: Any static host (Vercel, Netlify, Cloudflare Pages)
- Adapter: Vercel Node.js 20.x runtime
- Static file serving required for Web Worker scripts (`/workers/*.js`)
- HTTPS recommended for security headers

## Deployment

**Build Output:**
- Location: `apps/frontend/build/`
- Type: Static assets + Node.js serverless functions (when using Vercel)
- Size: Optimized with Terser minification

**Docker Support:**
- Base image: `oven/bun:1-alpine`
- Port: 5173
- Volume mounts: Source code, node_modules, .svelte-kit cache
- Development-focused (includes hot reload)
