  <action>
    First, install the file-type npm library (v19.7.1 - pure JS, works browser+Node):
    ```bash
    cd apps/frontend && bun add -D file-type
    ```

    Then create `apps/frontend/tests/fixtures/validators/magic-bytes.ts` with:

    1. Import fileTypeFromBuffer from 'file-type'

    2. Define MAGIC_SIGNATURES constant with Buffer signatures for formats file-type doesn't cover well:
       - PNG: [0x89, 0x50, 0x4e, 0x47, 0x0d, 0x0a, 0x1a, 0x0a]
       - JPEG: [0xff, 0xd8, 0xff]
       - WebP: RIFF header [0x52, 0x49, 0x46, 0x46] + check bytes 8-11 for 'WEBP'
       - TIFF LE: [0x49, 0x49, 0x2a, 0x00]
       - TIFF BE: [0x4d, 0x4d, 0x00, 0x2a]
       - BMP: [0x42, 0x4d]
       - GIF: [0x47, 0x49, 0x46, 0x38]
       - ICO: [0x00, 0x00, 0x01, 0x00]
       - PNM (PBM): [0x50, 0x31] or [0x50, 0x34] (ASCII P1/binary P4)
       - PNM (PGM): [0x50, 0x32] or [0x50, 0x35]
       - PNM (PPM): [0x50, 0x33] or [0x50, 0x36]
       - WAV: RIFF header + check bytes 8-11 for 'WAVE'
       - FLAC: [0x66, 0x4c, 0x61, 0x43]
       - MP3: [0xff, 0xfb] or [0xff, 0xfa] or [0x49, 0x44, 0x33] (ID3)
       - OGG: [0x4f, 0x67, 0x67, 0x53]
       - Opus: OGG container + 'OpusHead' at byte 28
       - PDF: [0x25, 0x50, 0x44, 0x46]
       - ZIP: [0x50, 0x4b, 0x03, 0x04] or [0x50, 0x4b, 0x05, 0x06] (empty)
       - 7Z: [0x37, 0x7a, 0xbc, 0xaf, 0x27, 0x1c]
       - TAR: check bytes 257-262 for 'ustar' (POSIX tar)
       - GZIP: [0x1f, 0x8b]
       - BZIP2: [0x42, 0x5a, 0x68]
       - XZ: [0xfd, 0x37, 0x7a, 0x58, 0x5a, 0x00]
       - XLSX/DOCX: ZIP signature (they're ZIP files)

    3. Export type ValidationResult:
       ```typescript
       interface ValidationResult {
         valid: boolean;
         detectedFormat: string | null;
         expectedFormat: string;
         confidence: 'high' | 'medium' | 'low';
         error?: string;
       }
       ```

    4. Create MagicByteValidator class with static methods:
       - validate(buffer: Buffer, expectedFormat: string): Promise<ValidationResult>
       - detectFormat(buffer: Buffer): Promise<string | null>
       - getSignature(format: string): Buffer | null

    The validate() method should:
    - First try file-type library for comprehensive detection
    - Fall back to manual signature check for formats file-type misses (PNM, TAR)
    - Handle text formats (txt, csv, json, xml, yaml, html, md) by checking valid UTF-8
    - Return confidence level: 'high' if file-type matches, 'medium' for manual check, 'low' for text
  </action>

       ```typescript

       import { MagicByteValidator, MAGIC_SIGNATURES } from './magic-bytes';
       ```

       Test cases:

       a) **Binary format detection** (one per category):
          - PNG: Create minimal PNG (8 bytes header), expect valid=true, detectedFormat='png'
          - JPEG: Create minimal JPEG header, expect valid=true
          - PDF: Create '%PDF-1.4' header, expect valid=true
          - ZIP: Create PK header, expect valid=true
          - FLAC: Create 'fLaC' header, expect valid=true

       b) **Compound signature formats**:
          - WebP: RIFF + WEBP at bytes 8-11, expect valid=true
          - WAV: RIFF + WAVE at bytes 8-11, expect valid=true

       c) **Text format detection**:
          - JSON: '{"key": "value"}', expect valid=true, confidence='low'
          - CSV: 'a,b,c\n1,2,3', expect valid=true
          - Invalid UTF-8: Buffer with 0xFF 0xFE bytes, expect valid=false for text formats

       d) **Format mismatch detection**:
          - PNG header but expect 'jpeg', expect valid=false, detectedFormat='png'
          - JPEG header but expect 'png', expect valid=false

       e) **Edge cases**:
          - Empty buffer: expect valid=false, error message
          - Buffer too short for signature: expect valid=false
          - Unknown format: expect detectedFormat=null

    Use minimal valid headers (not full files) for fast unit tests.
  </action>
